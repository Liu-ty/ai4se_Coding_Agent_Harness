## 9. Issues, Ambiguities, Conflicts

1. **Test stubs incomplete for ResolveStage:** The PLAN.md test examples for `TestResolveWindowsOverride` and `TestResolveLinuxOverride` omitted `Timeout` fields, causing `ResolveStage` to fail validation. Fixed by adding `Timeout: "1m"` to all test structs. This is a test-level gap in the plan, not a specification issue.

2. **No `git commit` performed:** Per experiment instructions, commits are deferred to the experiment verifier. This deviates from PLAN.md which specifies commit commands.

3. **Cross-host path validation:** Moving the Windows-path test to `config_windows_test.go` hid it on Linux and did not satisfy the requirement to reject Windows and POSIX absolute paths on every host. Independent remediation replaced it with platform-neutral cases and a host-independent validator.

4. **`default_profile` validation gap:** The review identified that `Load` accepted any string as `default_profile`. Added `ErrInvalidProfile` sentinel and validation. This was a plan-level omission — the PLAN.md did not explicitly require profile validation but the spec implies only valid profiles should be accepted.

## 14. Independent Verification and Remediation

Independent verification rejected the original unconditional `PASS` while preserving the useful Task 1 and Task 2 implementation under a student-approved moderate acceptance standard.

### 14.1 Public `Load` Contract

Added a compile-time assignment requiring:

```go
var _ func(io.Reader) (config.Config, error) = config.Load
```

Red evidence:

```text
cannot use config.Load (value of type func(r io.Reader) (*config.Config, error))
as func(io.Reader) (config.Config, error)
```

The implementation was then changed to return `Config`, and the focused contract test passed.

### 14.2 Host-Independent Absolute Paths

Added one platform-neutral test table covering:

- `C:\absolute\path`
- `C:/absolute/path`
- `\\server\share`
- `\rooted`
- `/absolute/path`

Red evidence:

```text
expected ErrAbsoluteWorkingDirectory for "\\rooted", got <nil>
```

The implementation now rejects host-native absolute paths, leading slash/backslash forms, drive-rooted paths, and UNC paths without depending on the current host OS. The Windows-only test file was removed.
